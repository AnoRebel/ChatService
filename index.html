System Breakdown

Abstract

In the recent years, building video applications on the web was a massive pain. Using Flash and proprietary codecs (which often required licensing), was a heavy burden. In the last few years, video chat technology has dramatically improved and Flash is no longer required. The video chat landscape is much simpler thanks to WebRTC: an open source project built and maintained by Google, Mozilla, Opera and others. WebRTC allows you to easily build real-time communication software in your browser and is being standardized at the W3C & IETF levels. Security wise, Auth0: a free authentication service, adds authentication, authorization, and user management to your web or mobile app, giving you more control buy giving you the option to choose who to allow access to.

Proposed System

In this proposed system the user will be able to open the door using both password and voice commands to ensure security and more enhancement on the system. In the system the user will type in the password first there after once near to the door or gate the user will give a voice command for the door to be opened or closed. If the user gives a close command the system will lock and does not require the user to type in the password to lock it the system will automatically lock to the password level.

Existing System

Traditional

In the past, video conferencing and/or chatting was a far-fetched idea. People needed to travel and meet to talk or have conferences.


Modern

The modern system allows video conferencing/chatting but with heavy PC load, laggy and bloated installation processes , and some don’t even support multi-user(conferencing).
